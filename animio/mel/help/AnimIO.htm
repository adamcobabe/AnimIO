Introduction...

The AnimIO tool is a quick way for the user to export and import animation.

It offers:

  • speed (millions of keys will be exported or imported within seconds)
  • animation export for selected nodes
  • animation export based on namespace (all animation connected to the namespace)
  • to replace or insert animation on import
  • to just import animation for special namespaces ( filtered import)
  • animation import at original time or current time
  • to add a prefix and multiple search and replace for import to fit the animation source to the current scene
  • to import just a timerange of the source animation
  • to import just a singel pose of the source animation
  • importing animation even without exporting first

Export:

The export layout contains:

  • mainly a textfield containing all the namespaces in scene and the option for selected nodes
    • marking a namespace here will tell the tool to export all the animation connected to this namespace when Export button is hit
    • the "selected nodes" text line will tell the tool to export all animation directly conected to whatever is selected in sceneat the moment the Export button is pressed
      • select, deselect and multiselect whatever you want to export
  • the Filetype switch tells the tool to export maya Ascii or maya Binary. Leave this on maya Ascii by default since it produces smaler files and is easier to debug
  • the Export... button will start the export process, a filebrowser will pop up, pointing to the right location allready, just enter a name that tells you about the content and thats it

Import:

The import layout contains:

  • the options: to switch between replace and insert
    • replace will replace the current animation by the imported animation for the animated timerange of the source
    • insert will insert the imported animation
  • the import at... switch sets the time the imported animation starts at
    • original time will import the animation exactly to where is starts in the source file (i.e. if exported animation starts at frame 1001 the imported also starts at 1001)
    • current time will import the animation starting at the timecursor position in timeline no matter what frame the animation starts in the source file
  • the load timerange options to import just parts or poses of the source animation
    • complete anim. will import the whole animated timerange of the source file
    • from file will try to load the timerange specified on the export or loads the complete anim. if not found
    • custom: offers the possibility to enter a start- and endframe of an individual timerange to import from the source animation
      • if start and end frame are the same it will import just the single pose of this frame
      • if just a startframe is entered the animation from this frame on to the end wil be imported
      • if just an endframe is entered the animation form the beginning to this endframe will be imported
      • when entering a reverse timerange (like start: 1050 and end: 1010) and import at... current time is set the import will make the current frame the last frame of the importet animation (just makes sense in replace mode)
    • last pose will import the last pose of the source animation, this is useful since you might not know the exact frame of the animations last pose.
    • first pose same as lastpose, but first pose will be loaded.
  • the add prefix: checkbox enables the possibility to enter a prefix in the textfield next to it
    • this text will be add to all animation nodes from the imported file (i.e. if you exported an animation from a character without namespace like "master_ctrl" and you want this animation to apply to you character in scene named "c_slim_rig:..." you can fix this by adding "c_slim_rig:" as a prefix and the animation will go to "c_slim_rig:master_ctrl")
  • the search checkbox enables the search & replace functionality
    • enter a search and a replace string into the according textfields and hit enter or the Add button and it will apear in the list below
    • all entries in the search & replace list will be treated serial from top to bottom (i.e. if you search for "slim" and replace it by "jalle" and in the next entry you replace "jalle" by "junior" it will also convert "slim" to "junior" since it was changed so jalle before, to avoid this think about the order first)
    • you can also use this field to apply something like a filter to your import, i.e: if you want to import every animation of a file except the camera - just use the search & replace feature to replace everything named "_cam" to something which is not in the scene for sure (like: _cam = xxxxx)
    • you can easily remove entries from the seach&replace textfield by marking them and hitting the remove selected button or just pressing del on you keyboard
  • the filtered input option will apply a filter to the animation imported
    • all animation imported needs to pass through the selected namespaces, this is a very easy way to just import one characters animation from a multicharacter other scene. For example with the settings above it will just import the animation to the dolly camera and junior, if it comes with the animation source file. You also should know that the filter is acting after the search & replace and adding the prefix (if used).
  • the Import... button will start the import process by opening a filebrowser pointing already to a sequence related folder, just select the animation file to import and thats it
Notice:
  • right now the import does not support undo so think of what you are doing first!
  • the default filtype of exported animation is .ani.ma or .ani.mb ...but since the exporter internally uses standart maya files you may also try to import the animation of a normal maya file (.ma or .mb) in case you forgot to export, this is fully suported and wanted and should work in almost every case except on characterSets, but be aware that the import will try to import all animation of the file, so if there are multiple characters in the source it will try to load all of them, except you use the filtered input option.
